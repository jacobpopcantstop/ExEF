Submission & Grading Pipeline

This area now frames submissions as a workflow with attachment metadata, release timing, and review history. File storage is still URL-based, but the record around each submission is much stronger.

Workflow

How the review queue should feel

  • Submit evidence once with a clean receipt trail
  • See whether the item is queued, under review, released, or needs revision
  • Know whether a missing purchase is blocking release
  • Watch module completion roll directly into capstone readiness

Today this still relies on account state and prototype-grade forms. The visual shell is being shaped toward a real reviewer portal.
