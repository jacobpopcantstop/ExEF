# Docs Guide

Active operating docs live at the top level of `docs/`.

Use these first:
- `progress.md` — canonical tracker for current launch/product work.
- `story-of-efi.md` — narrative history of the project, its phases, and what remains.
- `release-checklist.md` — deploy and release steps.
- `website-grade-rubric.md` — current grading rubric and score target.
- `deployment-baseline.md` — hosting and deployment baseline.
- `sales-conversion-audit.md` — current conversion-specific findings.
- `supabase-schema.sql` — reference schema snapshot.
- `data-retention-policy.md` — retention policy.
- `editorial-style-guide.md` — copy/style conventions.
- `video-pipeline.md` — video workflow notes.
- `api/openapi.yaml` — API contract.
- `assets/` — linked PDFs and supporting artifacts.

Historical audits, superseded plans, and old backlog docs were moved to `docs/outdated/`.
