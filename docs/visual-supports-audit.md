# Visual Supports Audit — Maximum Educational Impact

**Date:** 2026-03-13
**Scope:** All 29 educational and reference pages
**Current state:** 7 substantive visuals (SVGs in `/images/`) across the entire site

---

## Executive Summary

The site relies heavily on text to convey complex, abstract neuroscience
concepts — metaphors like "Air Traffic Control," "Wall of Awful," and "Leaky
Bucket" are described in paragraphs but never visualized.  Assessment tools
generate results programmatically but lack visual profiles (radar charts, bar
graphs).  Module 6 has **zero** visuals.  Adding 25-30 strategically placed
diagrams, flowcharts, and infographics would materially improve concept
retention, accessibility, and professional credibility.

## Implementation Notes

1. **Accessibility:** All visuals must include `alt` text, `aria-label`, and
   sufficient color contrast.  SVGs should use `<title>` and `<desc>` elements.
2. **Dark mode:** The site already has `dawson-thinking-doing-matrix-dark.svg`
   as precedent — all new SVGs should ship with dark-mode variants or use
   CSS `prefers-color-scheme` media queries.
3. **Existing SVG pipeline:** The `/images/` directory contains 8 SVGs, all
   following a consistent naming convention (`kebab-case.svg`).  New visuals
   should follow the same pattern.
4. **Assessment tool charts:** Chart.js or a lightweight charting library
   integrated via the existing JS pipeline would enable radar charts and
   bar graphs without heavy dependencies.
5. **Priority sequencing:** Start with Priority 1 items 1-6 (Modules 1-3),
   as these are the theoretical foundation that all later modules build on.
