EFI Capstone Transparency Rubric

Transparency sample for public review | Version 2026-02-16

This document shows how EFI distinguishes passing work from work that requires revision in the capstone review process. It is a public-facing standards document designed to clarify expectations before enrollment and submission.

Purpose Help candidates understand the scoring logic behind EFI's capstone review before formal submission. Use Of This Document Transparency reference only. It does not replace full reviewer scoring sheets or internal decision notes.
Passing Logic Passing requires acceptable performance across all required domains, not isolated strength in only one area. Review Notes Borderline or high-variance submissions may be escalated for additional manual review before final release.

Component 1: Intake Simulation

This component evaluates whether the candidate can create a strong first-session structure that is collaborative, strengths-based, and grounded in executive functioning language ordinary clients can understand.

Domain Passing Standard Needs Revision
Rapport Building
  • Uses primarily open-ended prompts and reflective listening.
  • Validates client frustration without judgment.
  • Maintains collaborative tone and clear role boundaries.
  • Uses mainly directive advice or closed questions.
  • Dismisses client affect and jumps too quickly to fixing.
  • Communicates in deficit-focused or corrective language.
EF Explanation (Psychoeducation)
  • Explains executive function using accessible, non-clinical language.
  • Uses clear process framing such as the air-traffic-control metaphor.
  • Preserves a strengths-based tone while translating behavior into skill language.
  • Uses dense jargon without translation.
  • Frames executive functioning problems as character flaws.
  • Omits practical behavior translation or client-relevant examples.

Component 2: Intervention Plan

This component evaluates whether the candidate can connect theory to intervention and design support structures that reduce friction at the point of performance.

Domain Passing Standard Needs Revision
Theoretical Grounding
  • Intervention choices map explicitly to Barkley, Brown, Ward, or Dawson/Guare concepts.
  • Rationale links each tactic to a documented executive functioning mechanism.
  • Suggestions are generic, motivational, or unsupported by a model.
  • No clear model-to-intervention rationale is provided.
Environmental Modification
  • Introduces concrete external supports before relying on willpower or insight alone.
  • Includes operational details tied to the point of performance.
  • Relies mostly on internal effort, motivation, or reminders to try harder.
  • Lacks actionable environmental changes or implementation detail.

Interpretation Notes

  • A passing result requires acceptable performance across all required domains.
  • High-variance or borderline submissions may be escalated to manual audit.
  • Final release follows EFI's delayed feedback and release workflow.
